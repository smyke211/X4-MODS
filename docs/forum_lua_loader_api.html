Lua Loader API

Author: SirNukes | Date: Mon, 12. Aug 19

Overview

Implements a generic method of loading custom lua files into X4, working around a bug in the intended method of loading lua code.

How to Use

In an MD script, add a cue following this template:

<cue name="Load_Lua_Files" instantiate="true">
  <conditions>
    <event_ui_triggered screen="'Lua_Loader'" control="'Ready'" />
  </conditions>
  <actions>
    <raise_lua_event name="'Lua_Loader.Load'" 
      param="'extensions.your_ext_name.your_lua_file_name'"/>
  </actions>
</cue>

Requirements

  • Replace "your_ext_name.your_lua_file_name" with appropriate path (no extension)
  • Lua file must be loose (not packed in cat/dat)
  • Extension can be ".lua" or ".txt" (txt may be needed for Steam Workshop)
  • Reload in-game with "/reloadui" in chat window

Dependency Loading

When loading completes, a message is printed to debuglog and a ui signal is raised.

Signal "control" field will be "Loaded " followed by the original param file path.

This can be used to set up loading dependencies:

<conditions>
  <event_ui_triggered screen="'Lua_Loader'" 
    control="'Loaded extensions.other_ext_name.other_lua_file_name'" />
</conditions>

How It Works

A small lua program provides two functions:

  1. Signal when it is loaded
  2. Receive file paths from MD scripts and load them using lua's "require"

The Problem It Solves

Standard ui.xml lua loading has issues:

  • Globals table is not initialized
  • No access to UI functions exported by other X4 lua files
  • No FFI support
  • No way to communicate with Mission Director
  • Basic lua functions unavailable

Prior workaround by morbideth was complex; this API provides simpler MIT-licensed solution.
