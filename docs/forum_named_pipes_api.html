[MOD/API] Named Pipes API

Author: SirNukes | Date: Mon, 25. Nov 19

Overview

Adds support for Windows named pipes for inter-process communication. Named pipes are OS level pseudo-files which support inter-process communication.

Pipes avoid the overhead of disk access from normal files (eg. debug logs), and are bidirectional.

Architecture: X4 acts as a client, with one or more external applications serving the pipes.

Components

  • Low level lua plugin + DLL — for pipe access
  • MD wrapper — interfaces between lua and MD pipe users
  • Optional Python external pipe host server — default server implementation

Requirements

  • Lua Loader API extension
  • Time API extension
  • Optionally: Simple Menu API
  • Python 3.6+ with pywin32 (if running from source) OR use prebuilt executable

Installation

  1. Place the named_pipes_api folder in extensions
  2. Place the X4_Python_Pipe_Server (exe or py) anywhere convenient
  3. Run pipe server alongside X4

API Layers

  • MD Pipe API — Named_Pipes.xml, allows MD code to read/write pipes
  • MD Server API — Pipe_Server_Host, allows MD to register Python modules with host server
  • Lua Pipe API — Named_Pipes.lua, lower level support
  • Winpipe DLL API — wraps Windows pipe functions (open, read, write, close, error check)

Key Features

  • Asynchronous callback-based model
  • Results delivered to callback cues
  • Timeout behavior configurable (default: disabled in v1.1+)
  • Option to flush pending accesses on timeout
