[MOD/API] Hotkey API

Author: SirNukes | Date: Mon, 25. Nov 19

Overview

Adds support for capturing key presses in X4, to implement custom hotkeys.

An external Python server is used for the key capture and combo recognition, and interfaces with X4 using named pipes.

Requirements

  • Named Pipes API (server should be running)
  • pywin32 and pynput packages (if using Python pipe server, not needed for prebuilt executable)

Usage — Register Action

<signal_cue_instantly cue="md.Hotkey_API.Register_Action" 
  param="table[
    $id = 'kc_target_follow',
    $cue = Target_Follow,
    $name = 'Follow Target',
    $description = 'Turns on autopilot to follow the target'
  ]"/>

Usage — Direct Key Mapping

<cue name="Register_Keys" instantiate="true">
  <conditions>
    <event_cue_signalled cue="md.Hotkey_API.Reloaded" />
  </conditions>
  <actions>
    <signal_cue_instantly cue="md.Hotkey_API.Register_Key" 
      param="table[$key='shift a', $id='kc_target_follow']"/>
  </actions>
</cue>

Contexts

Actions can specify valid contexts:

  • When flying
  • When walking
  • In any menu
  • In specifically named menus

Limitations

  • Windows backend confuses numpad enter and numpad / with non-numpad counterparts
  • Currently only supports keyboard inputs
