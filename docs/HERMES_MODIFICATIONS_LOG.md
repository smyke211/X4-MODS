## Файл
~/.hermes/SOUL.md

## До модификации
Пустой файл с комментарием (15 строк, только шаблон)

## После модификации
Добавлены 5 мандатных правил, применяемых КАЖДОЕ сообщение:

1. Стиль вывода (Output Efficiency):
   - Только русский язык
   - Пользователь видит только текстовый вывод
   - Короткие заметки на ключевых моментах
   - Саммери 1-2 предложения
   - Без комментариев по умолчанию

2. Скиллы (обязательная загрузка):
   - Загружай релевантные скиллы через skill_view ПЕРЕД ответом
   - Даже если думаешь что справишься без него
   - Ключевые скиллы перечислены

3. Безопасность (Executing actions with care):
   - Деструктивные операции → подтверждение
   - Один раз одобренное != одобрено везде
   - Не использовать деструктивные действия как shortcut

4. Параллельные вызовы:
   - Независимые — параллельно
   - Зависимые — последовательно

5. Память и поиск:
   - session_search ПЕРЕД вопросом пользователю
   - Устойчивые факты → memory
   - Процедуры → skills

## Изученные файлы
- ~/.hermes/config.yaml (377 строк) — полный конфигурационный файл
- ~/.hermes/SOUL.md — персона агента
- ~/.hermes/hermes-agent/AGENTS.md — документация разработчика
- ~/.hermes/hermes-agent/agent/prompt_builder.py (1084 строки) — сборка системного промта
- ~/.hermes/hermes-agent/run_agent.py — AIAgent класс, цикл разговора
- ~/.hermes/hermes-agent/hermes_state.py — SQLite + FTS5 сессий
- ~/.hermes/hermes-agent/agent/context_compressor.py — сжатие контекста
- ~/.hermes/hermes-agent/tools/registry.py — registry инструментов

## Найдено
1. Гермес — модифицируемый Python-проект
2. Процесс работает как: venv/bin/python -> /usr/bin/hermes
3. Структура проекта полная с тестами, gateway, TUI, ACP адаптером
4. Ключевые точки модификации:
   - agent/prompt_builder.py — сборка системного промта
   - hermes_state.py — поиск по сессиям (сейчас только FTS5)
   - agent/context_compressor.py — компактизация контекста
   - tools/registry.py — реестр инструментов
   - tools/code_execution_tool.py — execute_code песочница

## Выявленные ограничения
- `session_search` использует FTS5 (full-text search), не embeddings
- `memory` хранит плоские записи, нет графовых связей
- Скиллы загружаются по keyword matching, не семантически
- Компактизация контекста просто сжимает, не учитывает релевантность
- Нет фоновой координации контекста
